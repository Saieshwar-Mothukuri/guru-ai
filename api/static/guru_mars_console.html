Gap 5 — BMTC / Bus Fleet Integration (not yet built)

1,466 BMTC-related incidents = 30% of all events in this dataset. No link currently exists between GURU MARS alerts and BMTC operations. When a bus breakdown strands a major arterial, no dispatch signal reaches the BMTC control room. A bus fleet API layer (BMTC GTFS-RT feed → GURU MARS risk engine → operator pre-alert) is the single largest untapped integration and would give the recommendation engine ground truth on a cause type it currently treats as generic.

▸ Build notes & honest limitations
  • Weather modifier is illustrative, not derived from this dataset. The ASTraM export has no weather column (only 2 fog_low_visibility rows; 246 descriptions mention rain). The weather buttons apply fixed offsets (+12/+28/+22 pts) for demo purposes only — not computed from the CSV. Production upgrade: connect an IMD/OpenWeatherMap API and retrain Model 1 with weather as a real feature.
  • Mysore Road's real cascade partner is ORR East 1, not Magadi Road. The original concept assumed a Mysore Road→Magadi Road domino effect. Tested directly against the data at 15/30/60-minute and hourly granularity, that pair never clears r>0.12 — well short of significance. The two cascades that do hold up (shown live in Step 2 above) are Mysore Road→ORR East 1 and Bannerghata Road→IRR(Thanisandra road).
  • Officer IDs are real; names and ETAs are not. The dashboard shows real officer IDs from the dataset's assigned_to_police_id column (FKUSR01441 appears 52×, the most frequent). ETAs are placeholders — no ETA column exists in this export. Production reads live geo-tagged positions from GURU MARS's e-attendance feed.
  • Diversion routing isn't live. No MapmyIndia API key is available offline, so the diversion panel surfaces the cascade-predicted downstream corridor to avoid recommending, rather than a turn-by-turn route.
  • Zone & police-station fields are KNN-backfilled. ~58–69% of rows arrived with missing zone/junction. These are filled via a k-nearest-neighbours classifier on lat/lon, a reasonable proxy, not a true shapefile spatial join.
  • 3 rows of test data live in the raw export. A stray test_demo event_cause (3 of 8,173 rows) was found and excluded from the cause picker above as a data-quality artifact, not a real incident type.
  • Manpower unit counts are rule-based, not trained. No units_dispatched column exists in this dataset. The rule (base 1 unit, +1 for High/Critical risk, +1 if barricading, +1 for High priority, cap 5) is a defensible heuristic that can be replaced with a learned model the moment dispatch history exists.
